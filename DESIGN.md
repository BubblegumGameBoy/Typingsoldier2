# DESIGN.md — タイピング・レイド 設計 v1.0（実装済み）

> ステータス：**v1実装済み（index.html）**。バランス数値は初期値であり、プレイテストで調整する。

## 0. コンセプト

- **ベース**：ソルジャータイピング（Bubble-Wars）のタイピング＋撃破 ＋ spacebar-clicker の素材（クトゥルフ神格 webp / ユニット png / 背景）。
- **1プレイ＝60秒固定**。ドーパミンが立ち上がって消える前に終わる窓。小学生が1セッション耐えられる長さ。
- **ターゲット**：新規獲得より **“濃いリピーター”の定着**（小さく長く）。
- **トーン**：神話ホラーの渋い雰囲気（ダーク＋ゴールド＋明朝）。見た目は **A案（額縁リスキン）** を別途。

---

## 1. コアループ（1分モード＝α案）

1. **今日のボス**（クトゥルフ43体からローテ）が登場。
2. **60秒タイピング** → 入力ごとにボスへダメージ。
3. 累計ダメージが **討伐ライン(HP)** を超えた瞬間 → **討伐確定（＋ボーナス）**。ボスは断末魔だが、**60秒は最後まで殴れてダメージは加算され続ける**。
4. 60秒終了 → その回の **総ダメージ** が確定。
5. 討伐できた日は「**討伐！また明日**」→ **その日はロック**（翌日まで挑戦不可）。
6. 討伐前なら「**もう一回？**」で再挑戦（HPリセット＝ベスト更新狙い）。

各自が自分のボスと戦う **個人戦（B案）**。共有レイドは採らない（同時接続が薄い前提のため）。

---

## 2. なぜこの形か（設計意図）

**「救済（誰でもクリア）」と「実力差（ランキング）」を分離する**のが肝。

- 討伐ライン(HP)は **低め**にして “ほぼ全員が超えられる” → **討伐の達成感を全員に配れる**（小学生でもクリアできる）。
- ランキングは **60秒の総ダメージ（上限なし・オーバーキル込み）** → 速いタイパーほど上、天井なし。
- ※「討伐で即終了」にすると、倒した人全員がダメージ=HPで**同点**になり実力差が消える。だから **60秒は完走・オーバーキル加算**。

---

## 3. スコア／ランキング

- **日次ダメージ板**：今日のボスへの60秒ベストダメージ順。**日次で消えてOK**。
- **プロフィール（溜まる・retentionの本体）**：
  - 通算討伐日数 / 連続クリア日数（ストリーク） / 最高ダメージ。
  - ＝「何度もプレイしてる人の可視化」。“集めろ”と看板を張らず、数字だけ光らせる。
- 溜める対象を**プロフィールに逃がした**ので、「日次板は溜まらない」問題は解消。
  - 古参＝通算討伐の王座 ／ 後発＝ストリークで勝てる、の二軸で居場所を作る。

---

## 4. 日替わり＆公平性

- **ボス選定**：`dayIndex(JST) % ロスター長` で決定的。全員同じ。サーバ不要。
- **お題（単語の出題順）**：`dayIndex` をシードにした擬似乱数で固定 → 全員同じ出題順 → **ダメージ比較が公平**（Wordle型）。
- **リセット時刻**：日本時間0時で固定。

## 5. バランス設計 ★最重要（慎重に）

### 5.1 ダメージ計算（叩き台）

- 基本：**正解1文字ごとに `base` ダメージ**。
- **コンボ倍率**：連続正解で ×（カーブは要調整、**上限あり**）。
- **精度ペナルティ**：ミスでコンボ切れ＋わずかに減衰。
- （任意）**クリティカル**：一定確率で ×N。
- → 「速く・正確に・切らさず打つ」ほど伸びる。実力の主軸＝スループット×コンボ維持。

### 5.2 調整ノブと初期方針

| ノブ | 役割 | **実装値(v1)** |
|------|------|----------|
| **討伐ラインHP** | 誰がクリアできるか（救済の要） | **7,000**（全ボス共通）。ゆっくり打つ子（~1.2打/秒）でも1〜2回で届く想定 |
| 1打ダメージ | 基本火力 | **100** |
| コンボ倍率カーブ | 実力差の開き | **1打+2%、上限50コンボ＝×2.0**（青天井にしない） |
| 単語完成ボーナス | 打鍵の区切りの快感 | **+200** |
| 討伐ボーナス | 救済の底上げ | **+2,000 定額**（クリア/非クリアの段差のみに効く） |
| ボス別HP | 日毎の難度変化 | **全ボス共通**（毎日“誰でもクリア可”を担保） |
| ランク対象 | best / first / only | **その日のベスト1回**（リトライ自由） |
| クリティカル | 乱数演出 | **なし**（日替わりシード固定と相性の悪いRNGは排除＝公平） |

想定ダメージ帯（60秒・概算）：子供 ≈ 9,000〜11,000（クリア可能）／ 大人平均 ≈ 40,000 ／ 上級 ≈ 70,000+。約5〜7倍のスプレッドで実力差が出る。

### 5.3 バランスの落とし穴（常に意識）

- HPを実力者基準で上げると **子供がクリアできず離脱**＝目的の逆。**HPは“クリア線”、難度は“ダメージ量”で表現**する。
- 討伐ボーナスを大きくしすぎると実力ランキングが死ぬ。ボーナスは「クリア/非クリアの段差」だけに効かせる。
- リトライ無制限は子供の救済に必要だが、ランクが“回数ゲー”にならないよう **best-run** で受ける。
- **実数値はプレイテスト前提**。一発で決めない。ダメージ分布のログを見て寄せる。

## 6. データ構造（Firestore・既存を拡張）

```
// 日次スコア（既存 scores コレクションを拡張）
scores/{auto}
  uid, name, mode:"raid1min", boss, date(JST), damage, cleared(bool), ts

// プロフィール（新規）
players/{uid}
  name, totalClears, streak, lastClearedDate, bestDamage
```

- **身元**：Firebase 匿名認証（既にFirebase導入済み）。摩擦ゼロで永続uid。ロック判定＆プロフィール紐付けに使用。
- **日次ロック**：`players.lastClearedDate == 今日(JST)` で判定。
- ⚠️ ロックの強度は身元の強度と同じ（匿名認証は端末単位）。“濃い身内”前提なら十分。
